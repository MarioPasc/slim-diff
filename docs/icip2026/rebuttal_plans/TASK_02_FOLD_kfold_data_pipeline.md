# TASK 02 — 3-Fold Stratified Patient-Level Data Pipeline

## Context

SLIM-Diff trains on 78 epilepsy patients (all FCDII) from the Schuch et al. (2023) open-access dataset. The current codebase performs a single train/val/test split at the patient level (see `src/diffusion/data/splits.py`). Reviewer R1.3 requested stability assessment via multiple data partitions. The rebuttal commits to a **3-fold stratified evaluation** in the camera-ready version.

This task creates the fold-management infrastructure so that every downstream component (training, generation, evaluation) can operate on a specific fold without modification to those components.

## Your Deliverable

Implement a `KFoldManager` class and CLI utility that:
1. Creates 3 stratified folds at the **patient level** (no slice leakage)
2. Serialises fold assignments to a deterministic JSON file
3. Generates per-fold train/val/test CSV files compatible with the existing `create_dataloader()` function
4. Provides a Python API for other pipeline stages to query fold assignments

- `src/diffusion/data/splits.py` — **MODIFY**: add a `create_kfold_splits()` function that hooks into the existing `DatasetSplit` infrastructure

## Current Data Flow (Do Not Break)

The existing pipeline works as follows:

1. Raw MRI volumes are preprocessed and cached into a **slice cache** directory (`data/epilepsy/slice_cache/`) containing per-slice `.npz` files and a master CSV.
2. `splits.py` reads the cache CSV, groups slices by `subject_id`, and creates `train.csv`, `val.csv`, `test.csv` in the cache directory.
3. `dataset.py`'s `create_dataloader()` reads these CSVs to build PyTorch dataloaders.
4. The training runner (`runners/train.py`) calls `create_dataloader()`.

Your fold system must produce per-fold CSV files that are **drop-in replacements** for the existing `train.csv`, `val.csv`, `test.csv`. The training/generation pipeline then just needs to point to the correct fold directory.

### Splitting Strategy

Use `sklearn.model_selection.StratifiedKFold` with:
- `n_splits=3`
- `shuffle=True`
- `random_state=42`

Stratification variable: **binary pathology label** (has_lesion vs. no_lesion at the patient level). A patient is classified as "has_lesion" if they have ≥1 slice with `has_lesion=True` in the cache CSV.

Each fold produces a **test set** (the held-out fold) and a **train+val set** (the remaining two folds). The train+val set is further split 90/10 into train/val using a secondary stratified split with `random_state=42`.

### Patient-Level Integrity

**Critical**: All slices from a given patient must appear in the same split. Never split slices from the same patient across train/val/test. The stratification and splitting operates on the list of 78 `subject_id` values, then the slice-level CSVs are derived by filtering the master cache CSV by subject membership.

## Acceptance Criteria (Testable)

All tests must run without GPU (CPU-only, <1GB RAM).

### Test 1: No Patient Leakage
```python
def test_no_patient_leakage():
    """No patient appears in both train and test of any fold."""
    manager = KFoldManager(cache_dir=MOCK_CACHE, n_folds=3, seed=42)
    folds = manager.create_folds()
    
    for fold in folds:
        train_set = set(fold.train_subjects)
        val_set = set(fold.val_subjects)
        test_set = set(fold.test_subjects)
        
        assert train_set.isdisjoint(test_set), f"Fold {fold.fold_id}: train/test overlap"
        assert train_set.isdisjoint(val_set), f"Fold {fold.fold_id}: train/val overlap"
        assert val_set.isdisjoint(test_set), f"Fold {fold.fold_id}: val/test overlap"
```

### Test 2: All Patients Covered
```python
def test_all_patients_covered():
    """Every patient appears in exactly one test set across all folds."""
    manager = KFoldManager(cache_dir=MOCK_CACHE, n_folds=3, seed=42)
    folds = manager.create_folds()
    
    all_test = []
    for fold in folds:
        all_test.extend(fold.test_subjects)
    
    assert len(all_test) == len(set(all_test)), "Duplicate test subjects across folds"
    assert len(set(all_test)) == 78, "Not all patients covered"
```

### Test 5: Determinism
```python
def test_deterministic_folds():
    """Same seed produces identical fold assignments."""
    m1 = KFoldManager(cache_dir=MOCK_CACHE, n_folds=3, seed=42)
    m2 = KFoldManager(cache_dir=MOCK_CACHE, n_folds=3, seed=42)
    
    f1 = m1.create_folds()
    f2 = m2.create_folds()
    
    for a, b in zip(f1, f2):
        assert a.test_subjects == b.test_subjects
```

### Test 6: Slice Count Sanity
```python
def test_slice_counts_sum():
    """Total slices across train+val+test equals total slices in cache."""
    manager = KFoldManager(cache_dir=MOCK_CACHE, n_folds=3, seed=42)
    manager.write_fold_csvs()
    
    total_cache = len(pd.read_csv(MOCK_CACHE / "master.csv"))
    
    for fold_id in range(3):
        fold_dir = MOCK_CACHE / "folds" / f"fold_{fold_id}"
        n_train = len(pd.read_csv(fold_dir / "train.csv"))
        n_val = len(pd.read_csv(fold_dir / "val.csv"))
        n_test = len(pd.read_csv(fold_dir / "test.csv"))
        
        assert n_train + n_val + n_test == total_cache, \
            f"Fold {fold_id}: {n_train}+{n_val}+{n_test} != {total_cache}"
```

## Anti-Patterns — Do NOT:

- Do NOT modify `dataset.py` or `create_dataloader()`. Your output CSVs must be compatible as-is.
- Do NOT split at the slice level. Always group by `subject_id` first.
- Do NOT use random splitting without stratification. The pathology distribution must be balanced.
- Do NOT hardcode paths to the Picasso cluster. Use relative paths and config-driven paths.
- Do NOT delete or overwrite the original `train.csv`/`val.csv`/`test.csv`. They must remain for backward compatibility.
