# Study 1.2 — Distributional Fidelity via a Domain-Relevant Feature Extractor

## Critical framing — read first

**This is NOT a data-augmentation study.** It is a **distributional fidelity test** that uses a pretrained, real-data-trained segmentation model as a domain-relevant feature extractor — conceptually equivalent to KID using Inception features, but with a network that is sensitive to the lesion-anatomy structure we actually care about.

The protocol is **Train-Real-Test-Synthetic (TRTS)**, the inverse of the data-augmentation protocol (Train-Synthetic-Test-Real, TSTR). The question we answer is:

> *"Does the synthetic image-mask pair preserve the joint structure that a real-data-trained segmenter relies on?"*

If the segmenter's predictions on synthetic images match the synthetic ground-truth masks at a level comparable to its performance on real test data, then the synthetic distribution is faithfully aligned with the real distribution **at the level of features the segmenter exploits**. If not, there is a measurable joint-distribution gap.

This framing must be preserved in:
- The code (variable names, log messages, config keys).
- The output documentation (`RESULTS.md`).
- The eventual paper section title (suggested: *"Distributional Fidelity via a Domain-Relevant Reference Model"*, never *"Downstream Segmentation"* or *"Data Augmentation"*).

The TRTS protocol is established in the synthetic-data evaluation literature; cf. Esteban et al., *Real-valued (Medical) Time Series Generation with Recurrent Conditional GANs* (arXiv:1706.02633, 2017), §5; and the joint-synthesis evaluation discussion in Skandarani et al., *GANs for Medical Image Synthesis: An Empirical Study* (J. Imaging, 2023).

## Objective

Apply a **fixed, real-data-trained nnU-Net** (provided by a lab peer) as inference-only segmentation to the synthetic slices generated by each diffusion configuration. Compute Dice and HD95 between the segmenter's predictions on the synthetic image and the synthetic ground-truth mask. Compare across the three coupling configurations (independent / decoupled / shared) and against the segmenter's reference performance on real held-out slices.

The key metric is the **TRTS gap**:
$$\Delta_{\text{TRTS}} = \mathrm{Dice}_{\text{real}\to\text{real-test}} \;-\; \mathrm{Dice}_{\text{real}\to\text{synthetic}}$$

A small Δ_TRTS indicates that the synthetic distribution is indistinguishable from the real distribution from the segmenter's viewpoint.

### Backup model — Med-SAM2 (only if peer's nnU-Net is unavailable)

If the peer cannot deliver the nnU-Net within the 3-week window, the fallback is **Med-SAM2** (Ma et al., 2024, MONAI Model Zoo). Note carefully:

- Med-SAM2 is **promptable**, not category-specific. It requires either a bounding box or a point prompt at inference. For a fair TRTS test, supply the centroid of the synthetic GT mask as the prompt — this leaks the GT location to the model, so the test becomes a **shape-fidelity test conditional on location**, not a full TRTS test. Document this distinction explicitly.
- A pretrained, FCD-specific nnU-Net is strictly preferred. Use Med-SAM2 only if the peer fails to deliver.

A third fallback is the FCD-trained MONAI bundle from the model zoo if available; check `https://github.com/Project-MONAI/model-zoo/tree/main/models` for any FLAIR-trained brain-lesion bundles. Verify domain match before committing.

## Phase 1 — Implementation

### 1.1 Inference-only wrapper

Build `DomainReferenceSegmenter`, an inference-only wrapper around the pretrained nnU-Net. It must:

- Load weights from the peer-supplied bundle.
- Expose `.predict(image: torch.Tensor) -> torch.Tensor` producing a binary mask in `{0, 1}`.
- Apply the input pre-processing the segmenter was trained with (intensity normalisation, resampling if needed). **Mismatched preprocessing is the single most common failure mode in TRTS — verify against the peer's training config.**
- Apply the same intensity-domain conversion as the synthetic data: re-scale `[−1, +1] → [0, 1]` if the segmenter expects it.

Class signature:

```python
@dataclass(frozen=True)
class SegmenterConfig:
    bundle_path: Path
    expected_input_range: Tuple[float, float]  # (0., 1.) for nnU-Net standard
    target_spacing: Optional[Tuple[float, float]] = None  # if resampling needed
    device: str = "cuda"

class DomainReferenceSegmenter:
    def __init__(self, config: SegmenterConfig) -> None: ...
    def predict(self, image: torch.Tensor) -> torch.Tensor: ...
    def predict_batch(self, images: torch.Tensor) -> torch.Tensor: ...
```

### 1.2 TRTS evaluation pipeline

Build `compute_trts_metrics` that consumes a directory of synthetic NPZ files and produces per-slice Dice and HD95 between `segmenter.predict(synthetic_image)` and `synthetic_GT_mask`.

```python
@dataclass
class TRTSResult:
    config_name: str           # "independent" | "decoupled" | "shared"
    fold: int
    dice_per_slice: np.ndarray
    hd95_per_slice: np.ndarray
    n_slices: int
    n_empty_gt_masks: int      # for transparency
```

Aggregation:
- Mean and std Dice over non-empty-GT slices.
- HD95 over non-empty-GT slices (empty masks → undefined; report n_empty separately).
- A per-z-bin breakdown (matching the existing per-z-bin analysis style of Figure 2 in the manuscript) so we can show whether fidelity degrades at extreme axial positions.

### 1.3 Reference: real test-set baseline

Run the **same** segmenter on the real held-out test slices to establish the reference Dice. This is the upper bound — synthetic configurations cannot exceed this in a properly aligned distribution; if any do, that is a sign of an unrealistically simple synthetic distribution (lesions placed in canonical locations, etc.) and should be noted.

### 1.4 Statistical comparison

Per-fold Dice values for each configuration, compared via:
- **Friedman test** across the four conditions (real-test reference, independent, decoupled, shared) on paired folds.
- **Nemenyi post-hoc** if significant.
- **Cliff's δ** between each synthetic configuration and the real-test reference, to quantify the magnitude of the TRTS gap.

The pre-registered hypothesis is:
$$\Delta_{\text{TRTS}}^{\text{shared}} \;\le\; \Delta_{\text{TRTS}}^{\text{decoupled}} \;\le\; \Delta_{\text{TRTS}}^{\text{independent}}$$

## Compute budget

- Inference is **forward-only** — no training. Estimated cost: ~30–60 minutes per 36k slices on a single A100, depending on whether the segmenter is 2D or 2.5D.
- Total: 4 conditions × 3 folds × ~36k slices each ≈ **~6–12 hours of inference**.
- This is by far the cheapest of the three studies (1.1, 1.2, 2.1) in compute terms, but the most fragile in setup terms because of preprocessing mismatch risk.

## Risk register

- **R1: Peer cannot deliver pretrained nnU-Net in time.** Mitigation: fall back to Med-SAM2 with centroid prompts (documented as a different test). Communicate the fallback transparently in the paper. Worst case: skip this study and rely on Tiers 1.1 + 2.1 alone.
- **R2: Preprocessing mismatch causes the segmenter to fail catastrophically on synthetic images even though they are visually plausible.** This is the highest-probability failure. Mitigation: (a) verify synthetic intensity histogram against the segmenter's training-set histogram; (b) test on real held-out slices first to confirm the segmenter loads correctly; (c) inspect 10 random synthetic slices visually before launching full inference.
- **R3: Synthetic configurations all achieve Dice within noise of the real reference.** This is a **strong positive result**, not a problem — it means joint synthesis preserves segmenter-relevant structure. Frame accordingly.
- **R4: Synthetic Dice exceeds real-reference Dice.** This indicates an over-simplified synthetic distribution (lesions in canonical locations, lower morphological diversity). Investigate by comparing synthetic and real lesion-position distributions. If confirmed, report it as a limitation, not a strength.
- **R5: Severe per-z-bin degradation at extreme axial positions.** Expected and consistent with the data scarcity story; report it but do not over-interpret.

## Out of scope

- **Training the segmenter**. We are inference-only by design.
- **Hyperparameter tuning of the segmenter** (test-time augmentation, ensembling, threshold tuning). Use the segmenter as the peer delivers it; tuning would invalidate the comparison.
- **Adding synthetic to real for retraining** (this would be the data-augmentation experiment we are explicitly avoiding).
- Cross-modality TRTS (e.g. T1 segmenter applied to FLAIR synthetic). Out of scope for this paper.
